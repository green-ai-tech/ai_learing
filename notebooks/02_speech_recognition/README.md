## 🎯 核心学习路径

### 第一阶段：音频信号处理
- **内容**: 音频文件的读取、采样率转换、特征提取。
- **重点**:
    - **音频格式**: 理解 WAV, MP3 等格式的底层表示。
    - **特征提取**: 梅尔频率倒谱系数 (MFCC)、频谱图 (Spectrogram)、梅尔频谱图 (Mel-Spectrogram)。
    - **预处理**: 音频降噪、端点检测 (VAD)、标准化。

### 第二阶段：语音识别基础
- **内容**: 将音频信号转换为文本 (Speech-to-Text, STT)。
- **重点**:
    - **声学模型**: 使用深度学习模型（CNN, RNN, Transformer）从音频特征预测音素或字符。
    - **语言模型**: 结合上下文信息，提高识别准确率。
    - **解码器**: 将模型输出转换为最终文本（CTC, Beam Search）。

### 第三阶段：语言识别与进阶应用
- **内容**: 判断一段音频属于哪种语言，以及更复杂的语音任务。
- **重点**:
    - **语言分类**: 使用分类模型识别音频的语言类别（如英语、中文、法语等）。
    - **端到端模型**: 学习现代端到端语音识别架构（如 Whisper, Wav2Vec2）。
    - **实战训练**: 通过 `train_01.py`, `train_03.py` 等脚本掌握完整的训练流程。

## 💡 学习建议

1. **理解音频本质**: 音频是一维时序信号，与图像的二维空间信号处理方式完全不同。重点掌握时域和频域分析。
2. **特征工程很重要**: 深度学习的性能高度依赖输入特征。MFCC 和 Mel-Spectrogram 是语音任务中最常用的特征。
3. **对比视觉与语音**: 思考 CNN 在图像（2D 卷积）和语音（1D 卷积或 2D 频谱图卷积）中的应用差异。
4. **关注数据长度问题**: 语音识别的难点在于输入（音频帧数）和输出（文本字符数）长度不一致，理解 CTC 或 Attention 如何解决这个问题。
